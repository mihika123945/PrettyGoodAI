# Five-Minute Walkthrough Outline

1. **Problem and result (30 seconds):** show the challenge and one strong example recording.
2. **Architecture (60 seconds):** SignalWire call -> cXML media stream -> Gemini patient bot -> artifacts.
3. **Natural conversation (60 seconds):** play turn-taking, correction, and interruption moments.
4. **Testing strategy (60 seconds):** show scenario diversity and why the edge cases matter.
5. **Findings and iteration (60 seconds):** connect a useful bug to transcript/audio evidence, then show one improvement made after an early call.
6. **Run command and repository (30 seconds):** demonstrate dry-run safety and artifact layout.

For the separate AI-debugging recording, capture a real failure, the exact prompt given to the AI, the proposed diagnosis, the code diff, and the verification command. Good examples from this project: SignalWire migration, ngrok auth/version issues, recording download fallback, or Gemini quota handling.
